# BTCUSDT 永续合约纸面策略说明

> 更新时间：2026-07-13  
> 当前策略：`scalp-tpsl-perp-paper`  
> 当前模式：paper trading，本地模拟成交，不是真实交易所实盘下单。
> v2 更新：已加入趋势强度、确认 K 线、ATR 波动率、成交量过滤，以及按风险比例计算 paper 下单数量。

## 1. 当前实际参数

| 项目 | 当前值 | 说明 |
| --- | --- | --- |
| 市场 | `perpetual` | USDT 本位永续合约 |
| 交易对 | `BTCUSDT` | 当前只保留合约策略测试 |
| 周期 | `1m` | 使用 1 分钟 K 线 |
| 策略类型 | `scalp-tpsl` | 短线均线 + 固定止盈止损 |

| 止损 | `0.2%` | 开仓价上下 0.2% |
| 冷却 | `1` 根 K 线 | 平仓后至少等 1 根 K 线 |

## 2. 策略目前有哪些功能

当前已经支持：

- 读取本地真实 Binance 合约 1m K 线。
- 读取合约 mark price，并在看板里用标记价估算实时持仓盈亏。
- 允许合约开多。
- 允许合约开空。
- 每次开仓自动写入止盈价和止损价。
- 每轮策略运行前先检查已有持仓是否触发止盈/止损。
- 每轮策略运行前会检查趋势反转，多单遇到 `fast < slow`、空单遇到 `fast > slow` 会主动平仓。
- paper 未实现盈亏和已实现盈亏已按开仓、平仓双边手续费/滑点估算。
- 开仓前可以过滤弱趋势、低波动、异常高波动、低成交量信号。
- 可以使用 `risk_pct` 按账户权益和止损距离动态计算数量，并用 `max_notional_pct` 限制最大名义仓位。
- 同一策略同一交易对只允许一个打开的纸面持仓。
- 每轮都保存信号、订单、持仓、账户、回测和绩效快照。
- 看板默认展示合约 `BTCUSDT / 1m`，并过滤掉现货交易噪音。

当前没有支持：

- 没有减仓。
- 没有分批止盈。
- 没有移动止损。
- 没有追踪止盈。
- 没有加仓。
- 没有反手开仓。例如多单止损后，不会在同一轮马上反手开空。
- 没有真实交易所下单。
- 没有真实账户保证金、强平价、资金费率结算。
- 没有多人资金份额、申购赎回、跟单滑点分摊和业绩费结算。

## 3. 什么时候开多

当没有打开的纸面持仓时，满足以下条件才开多：

1. 最近 3 根 K 线收盘均价 > 最近 9 根 K 线收盘均价。
2. 最新收盘价 > 上一根收盘价。
3. 当前不在冷却期。

开多后：

- 订单方向：`buy`
- 持仓方向：`long`
- 数量：`0.01 BTC`
- 止盈价：开仓价 * `1.0035`
- 止损价：开仓价 * `0.998`

示例：

如果开多价是 `63900`：

- 止盈价 = `63900 * 1.0035 = 64123.65`
- 止损价 = `63900 * 0.998 = 63772.2`

## 4. 什么时候开空

当没有打开的纸面持仓，并且市场是 `perpetual` 时，满足以下条件才开空：

1. 最近 3 根 K 线收盘均价 < 最近 9 根 K 线收盘均价。
2. 最新收盘价 < 上一根收盘价。
3. 当前不在冷却期。

开空后：

- 订单方向：`sell`
- 持仓方向：`short`
- 数量：`0.01 BTC`
- 止盈价：开仓价 * `0.9965`
- 止损价：开仓价 * `1.002`

示例：

如果开空价是 `63748.1`：

- 止盈价 = `63748.1 * 0.9965 = 63524.98165`
- 止损价 = `63748.1 * 1.002 = 63875.5962`

## 5. 什么时候平仓

当前 paper trading 实时循环按以下顺序检查平仓：

1. 先检查固定止盈/止损。
2. 如果没有触发止盈/止损，再检查趋势反转。
3. 触发后整仓关闭，暂不做分批减仓。

### 多单平仓

多单触发条件：

- 当前价格 >= 止盈价：触发 `take_profit`
- 当前价格 <= 止损价：触发 `stop_loss`
- 快均线 < 慢均线：触发 `trend_reversal`

多单盈亏计算：

```text
毛盈亏 = (当前价格 - 开仓价) * 数量
净盈亏 = 毛盈亏 - 开仓名义价值 * (手续费 + 滑点) - 当前名义价值 * (手续费 + 滑点)
```

### 空单平仓

空单触发条件：

- 当前价格 <= 止盈价：触发 `take_profit`
- 当前价格 >= 止损价：触发 `stop_loss`
- 快均线 > 慢均线：触发 `trend_reversal`

空单盈亏计算：

```text
毛盈亏 = (开仓价 - 当前价格) * 数量
净盈亏 = 毛盈亏 - 开仓名义价值 * (手续费 + 滑点) - 当前名义价值 * (手续费 + 滑点)
```

## 6. 回测和平仓逻辑对齐状态

回测函数和实时 paper 现在都包含固定止盈止损和趋势反转平仓。

多单：

- 当前价格 >= 止盈价：平仓。
- 当前价格 <= 止损价：平仓。
- 快均线 < 慢均线：平仓。

空单：

- 当前价格 <= 止盈价：平仓。
- 当前价格 >= 止损价：平仓。
- 快均线 > 慢均线：平仓。

注意：回测在历史 K 线里用下一根价格模拟趋势反转后的成交；实时 paper 无法知道下一根价格，所以用当前轮最新价格平仓。这一点仍会造成轻微口径差异，但退出规则已经一致。

## 7. 什么时候减仓

目前没有减仓逻辑。

当前实现是：

1. 没仓时，满足信号后一次性开 `0.01 BTC`。
2. 有仓时，不再开新仓。
3. 到止盈或止损时，整仓关闭。

也就是说，现在是固定仓位、整进整出。

## 10. 当前策略风险和待完善点

当前策略偏简单，适合先做 paper trading 跑通链路，但不适合直接大资金实盘。

建议后续优先补：

- 减仓：例如盈利到 0.2% 先平一半，剩余仓位移动止损。
- 移动止损：盈利后把止损移动到开仓价附近，减少回撤。
- 反手规则：止损后是否允许反向开仓，需要独立判断，不能盲目反手。
- 波动率过滤：波动过小时不交易，波动过大时降低仓位。
- 成交量过滤：只在成交量放大时开仓。
- 资金费率过滤：资金费率异常时暂停开多或开空。
- 最大连续亏损熔断：连续亏损达到阈值后暂停策略。
- 真正 reduce-only 平仓单：接真实交易所时必须防止平仓单变成反向开仓。

## 10.1 面向跟单资金的底线

如果后续有多人申请加入资金，这个系统不能只看“策略信号”。必须把它当成资金池/跟单系统来设计：

- 不承诺持续盈利，只展示真实历史收益、回撤、胜率、亏损周期和成本。
- 每个跟单账户必须按权益比例独立算仓位，不能所有资金共用固定 `0.01 BTC`。
- 必须有最大日亏损、最大周亏损、最大连续亏损暂停。
- 必须有最大资金容量，资金太大后同一信号会产生更大滑点。
- 必须记录每笔交易的开仓原因、平仓原因、净盈亏、手续费、滑点和跟单账户分摊结果。

当前 v2 只完成了第一步：策略侧支持 `risk_pct` 和 `max_notional_pct`，让仓位可以随权益变化。资金份额、跟单队列、业绩报表和合规模块还没做。

## 11. 仔细分析后的核心问题

当前策略最大的问题不是“不能开多开空”，而是信号、成本、退出和风控都还比较粗。它已经能跑通合约 paper trading 链路，但离可以稳定评估收益还有距离。

### 11.1 净盈亏比被手续费和滑点吃掉

当前参数：

- 止盈：`0.35%`
- 止损：`0.2%`
- 手续费：`0.0005`
- 滑点：`0.0005`
- 回测单边成本：`0.1%`
- 一进一出成本：约 `0.2%`

表面看：

```text
毛止盈 / 毛止损 = 0.35 / 0.2 = 1.75
```

但扣掉一进一出成本后，大致变成：

```text
净止盈 ≈ 0.35% - 0.2% = 0.15%
净止损 ≈ 0.2% + 0.2% = 0.4%
净盈亏比 ≈ 0.15 / 0.4 = 0.375
```

这意味着策略必须有很高胜率才可能覆盖成本。对 1m 短线来说，这个要求偏苛刻。

优化方向：

- 要么扩大止盈目标，例如 `0.6%` 到 `1.0%`；
- 要么降低交易成本假设；
- 要么只在波动率足够大时开仓；
- 要么用分批止盈 + 移动止损保护利润。

### 11.2 1m SMA 3/9 太容易被噪音触发

当前开仓条件只看：

- `3` 根均线和 `9` 根均线的大小关系；
- 最新收盘价是否比上一根高或低。

这个信号非常敏感，在横盘震荡里容易反复开仓、止损、再开仓。特别是在 BTC 1m K 线上，很多价格变化只是噪音，不代表真实趋势。

优化方向：

- 增加趋势强度过滤，例如均线差距必须超过 `0.05%` 或 `0.1%`。
- 增加多周期过滤，例如 1m 开仓必须顺着 5m 或 15m 方向。
- 增加成交量过滤，只在成交量高于最近 N 根均量时开仓。
- 增加波动率过滤，ATR 太小时不交易，ATR 太大时降低仓位。

### 11.3 实时 paper 和回测退出逻辑已经基本对齐

当前回测和实时 paper 都有趋势反转平仓：

- 多单遇到 `fast < slow` 可以平仓；
- 空单遇到 `fast > slow` 可以平仓。

剩余差异在成交价口径：

- 回测趋势反转用下一根 K 线价格模拟成交；
- 实时 paper 用当前轮最新价格平仓；
- 这更接近实时系统能做到的行为，但和历史回测仍不会完全一模一样。

优化方向：

- 后续可以给实时 paper 增加更明确的成交模型，例如用 mark price、bid/ask 或下一轮确认价。
- 可以在 trade log 中记录 `exit_reason=trend_reversal` 和实际成交价，便于复盘。

### 11.4 看板盈亏已经计入手续费和滑点，但还没计入资金费率

当前 paper 持仓盈亏按净盈亏估算：

```text
净盈亏 = 毛盈亏 - 开仓成本 - 平仓成本
成本 = 名义价值 * (手续费 + 滑点)
```

现在 realized PnL 和 unrealized PnL 都会扣除开仓、平仓两边的手续费/滑点估算。剩余差距主要是：

- 还没有资金费率成本；
- 还没有真实 bid/ask spread；
- 还没有真实成交失败、部分成交、盘口冲击。

优化方向：

- 合约持仓增加资金费率成本字段。
- 行情同步增加 best bid/ask，用于更真实地估算成交和滑点。

### 11.5 当前仓位固定，不按风险动态调整

当前每次固定开 `0.01 BTC`。这很简单，但不够稳健。

问题：

- BTC 价格变化后，名义仓位也变化；
- 止损距离固定百分比时，单笔风险还算可控；
- 但如果后续改成 ATR 止损，固定数量就不合理；
- 资金变大或变小时，仓位不会自动适配。

优化方向：

```text
下单数量 = 账户权益 * 单笔风险比例 / abs(开仓价 - 止损价)
```

例如单笔最多亏 `0.3%` 账户权益，再根据止损距离反推出数量。

### 11.6 没有资金费率、价差、强平距离保护

当前是合约策略，但合约特有风险还没有真正进入策略判断：

- 资金费率过高时，做多或做空成本会变大；
- mark price 和 last price 可能短时偏离；
- 高杠杆时需要关注强平距离；
- 真实交易所平仓必须使用 reduce-only 防止反向开仓。

当前杠杆是 `1x`，所以强平风险较低。但如果后续提高到 `2x` 或 `3x`，这些保护必须先补。

## 12. 建议的优化优先级

### P0：先修正评估口径

这些属于必须先做，否则后续参数优化容易误判。

| 优化项 | 原因 | 建议 |
| --- | --- | --- |
| 统一回测和 paper 退出逻辑 | 已完成基础对齐 | 后续细化成交价模型 |
| paper 盈亏扣除手续费/滑点 | 已完成基础估算 | 后续加入资金费率和 bid/ask spread |
| 记录每笔平仓原因 | 后续分析必须知道是止盈、止损还是趋势反转 | `paper_positions` 增加 close_reason 或另建 trade log |
| 策略参数版本化 | 方便对比不同参数结果 | 在 run config 中记录完整参数和版本 |

### P1：提高信号质量

当前信号太简单，容易在震荡里频繁亏损。

| 优化项 | 推荐实现 |
| --- | --- |
| 均线差过滤 | 已支持：`min_trend_spread_pct` |
| 多周期趋势过滤 | 1m 开仓方向必须和 5m/15m 趋势一致 |
| 成交量过滤 | 已支持：`volume_window` + `min_volume_ratio` |
| 波动率过滤 | 已支持：`atr_window` + `min_atr_pct` + `max_atr_pct` |
| 确认 K 线 | 已支持：`confirm_bars` |
| 冷却加强 | 止损后冷却 3 到 5 根 K 线，止盈后冷却 1 到 2 根 |

### P2：优化退出策略

固定 TP/SL 太硬，容易出现“刚盈利又回撤止损”。

建议顺序：

1. 盈利到 `0.2%` 时平掉 `50%`。
2. 剩余仓位止损移动到开仓价附近。
3. 盈利继续扩大时使用移动止损。
4. 均线反转时不等待止损，主动平仓。

可以先用以下规则：

```text
多单：
- +0.2%：减仓 50%，止损移动到开仓价
- +0.5%：启动 trailing stop，回撤 0.2% 平剩余
- -0.2%：整仓止损

空单：
- -0.2% 方向盈利：减仓 50%，止损移动到开仓价
- -0.5% 方向盈利：启动 trailing stop，反向回撤 0.2% 平剩余
- +0.2% 方向亏损：整仓止损
```

### P3：合约专用风控

在 1x paper 阶段可以先记录，真实交易前必须实现。

| 风控项 | 建议 |
| --- | --- |
| 最大日亏损 | 当日亏损达到 `1%` 到 `2%` 暂停策略 |
| 连续亏损 | 连续 `3` 笔亏损暂停 30 到 60 分钟 |
| 最大仓位 | 单策略名义仓位不超过权益的 `1x` 到 `2x` |
| 资金费率 | 资金费率绝对值过高时不开仓 |
| 强平距离 | 小于安全阈值时禁止加仓或开仓 |
| reduce-only | 所有平仓单必须 reduce-only |

## 13. 参数优化建议

当前参数不建议直接认定为有效参数，只能算跑通链路的初始参数。

### 13.1 可以优先测试的参数组合

| 组合 | fast | slow | TP | SL | 适用假设 |
| --- | --- | --- | --- | --- | --- |
| 当前基准 | 3 | 9 | 0.35% | 0.2% | 交易频率高，但成本压力大 |
| 降噪版本 | 5 | 20 | 0.6% | 0.3% | 减少噪音交易 |
| 趋势版本 | 9 | 30 | 0.8% | 0.4% | 更少交易，持仓更久 |
| 快速试错 | 3 | 12 | 0.5% | 0.25% | 保留短线频率，略提高净收益空间 |

参数评估不能只看收益率，还要同时看：

- 交易次数是否足够；
- 胜率；
- 平均盈利；
- 平均亏损；
- 手续费占毛利润比例；
- 最大连续亏损；
- 最大回撤；
- 多空分别表现；
- 不同日期区间是否稳定。

### 13.2 不建议的优化方向

短期不建议做：

- 不建议直接加到高杠杆。
- 不建议马丁加仓。
- 不建议亏损后翻倍开仓。
- 不建议在没有 reduce-only 保护前接真实合约平仓。
- 不建议只根据最近几十根 K 线就判断策略有效。
- 不建议只优化某一天的数据，否则容易过拟合。

## 14. 建议实施顺序

建议按下面顺序做，避免一次改太多导致无法判断效果。

1. 已完成：对齐回测和 paper 退出逻辑，paper 支持趋势反转平仓。
2. 已完成：paper PnL 加入手续费和滑点，避免看板盈亏过于乐观。
3. 已完成：增加均线差、确认 K 线、成交量、ATR 过滤。
4. 已完成：增加 `risk_pct` 和 `max_notional_pct`，支持按风险比例计算 paper 仓位。
5. 增加 trade log，记录每次开仓、平仓、原因、持仓时长、净盈亏。
6. 改固定 TP/SL 为 ATR 或半动态 TP/SL。
7. 增加分批止盈和保本止损。
8. 增加连续亏损暂停和日亏损暂停。
9. 做参数网格回测和 walk-forward 验证。
10. 连续 paper trading 至少 2 到 4 周，再考虑小资金真实灰度。

## 15. 下一版建议目标

下一版策略可以命名为：

```text
scalp-tpsl-v2
```

建议目标：

- 保留当前 1m 合约短线方向；
- 开仓增加均线差、成交量、波动率过滤；
- 平仓统一支持止盈、止损、趋势反转；
- PnL 计入手续费和滑点；
- 支持部分止盈和保本止损；
- 看板展示每笔交易的净盈亏和退出原因。

这版完成后，再评估是否需要提高交易频率或提高杠杆。当前阶段优先目标不是更激进，而是先让每笔交易的风险收益结构变合理。
