# 合约趋势短线策略增强方案

本文基于当前 `perp-trend-scalp-v2-paper` 策略做整体检查，重点覆盖趋势理解、单边行情处理、防爆仓和后续跟单产品风险控制。

结论先说清楚：当前策略已经不是纯固定仓位策略，已经有动态止盈止损、风险仓位、强平距离和资金费率过滤。但它还不能算完整的趋势系统，更准确是“短周期均线动量 + ATR 止盈止损 + paper 风控仓位”的合约模拟策略。要面向真实交易和跟单资金，必须先增强趋势分层、账户级熔断和原生保护单确认。

## 0. 当前落地状态

截至当前版本，v8 已完成第一阶段：

- 已实现：`15m + 1h` 高周期趋势过滤，默认开启；两个周期 EMA 快慢线方向一致才允许新开仓。
- 已实现：保本止损，浮盈达到 `1R` 后把止损移动到手续费调整后的保本价。
- 已实现：ATR trailing stop，浮盈达到 `1.5R` 后按 `1.2 * ATR` 追踪止损，只向更保护的方向移动。
- 已实现：`paper_positions.close_reason` 入库，平仓原因可记录为 `take_profit`、`stop_loss`、`trend_reversal`。
- 已实现：日内已实现 PnL 和连续亏损次数接入 `risk.AccountSnapshot`，可触发 `daily_loss_halt` 和 `consecutive_loss_halt`。
- 已实现：`cmd/papertrade` 已拆分为入口、配置、运行循环、持仓风控、行情和信号趋势等文件，降低单文件复杂度。
- 已实现：本地启动脚本会同步执行周期、`15m` 和 `1h` K 线，避免趋势数据缺失。
- 未实现：分批止盈和 follower 独立风控，仍在后续阶段。

## 1. 当前策略现状

### 1.1 趋势理解

当前入场主要看短周期 SMA：

- 默认周期是 `5m`，快线 `3`，慢线 `9`。
- `fast > slow` 且最近 K 线方向确认时开多。
- `fast < slow` 且允许做空时开空。
- 额外过滤包含均线差、ATR 波动区间、成交量比例、追价距离和近期回踩。

对应代码：

- `cmd/papertrade/main.go:27` 定义默认周期、快慢线、动态 TP/SL 和信号过滤参数。
- `internal/backtest/model.go:494` 计算短周期快慢均线信号。
- `internal/backtest/model.go:501` 过滤均线差过小的震荡信号。
- `internal/backtest/model.go:507` 过滤 ATR 过低或过高的行情。
- `internal/backtest/model.go:519` 过滤成交量不足的信号。
- `internal/backtest/model.go:525` 到 `internal/backtest/model.go:541` 决定开多或开空。

问题是：这里的“趋势”只来自当前交易周期，缺少 `15m / 1h` 更高周期确认。它能识别局部动量，但不能判断当前到底是大级别单边上涨、单边下跌，还是震荡假突破。

### 1.2 平仓逻辑

当前已有三类退出：

- 止盈：价格触达持仓记录里的 `take_profit_price`。
- 止损：价格触达持仓记录里的 `stop_loss_price`。
- 趋势反转：持仓期间快慢均线反向时退出。

对应代码：

- `cmd/papertrade/main.go:1326` 到 `cmd/papertrade/main.go:1345` 按多空方向判断 TP/SL。
- `cmd/papertrade/main.go:1347` 到 `cmd/papertrade/main.go:1365` 按快慢均线反向判断趋势退出。
- `internal/backtest/model.go:253` 到 `internal/backtest/model.go:276` 在回测里对齐 TP/SL 和趋势反转退出。
- `internal/backtest/model.go:546` 到 `internal/backtest/model.go:566` 回测中同一根 K 线同时碰到止盈止损时优先按止损处理，逻辑偏保守。

问题是：趋势退出仍然只看短周期快慢线。在强单边行情里，短周期回踩仍然可能把仓位洗出去。当前已补保本止损和 ATR trailing stop，但还缺分批止盈、趋势未破时的剩余仓位持有规则和回踩再入场机制。

### 1.3 仓位和防爆仓

当前开仓数量已经按止损距离反推：

```text
风险预算 = 账户权益 * risk_pct / 100
下单数量 = 风险预算 / abs(开仓价 - 止损价)
```

同时会被名义价值、保证金、可用余额和最小下单数量继续压缩。

对应代码：

- `cmd/papertrade/main.go:1432` 到 `cmd/papertrade/main.go:1468` 按止损距离计算数量。
- `cmd/papertrade/main.go:1550` 到 `cmd/papertrade/main.go:1582` 用名义价值、保证金和可用余额压缩数量。
- `cmd/papertrade/main.go:861` 开仓前估算强平价。
- `internal/risk/model.go:190` 到 `internal/risk/model.go:219` 检查杠杆、保证金、强平距离和资金费率。

当前默认风控配置：

- `max_order_risk_pct=1`
- `max_initial_margin_pct=35`
- `max_available_balance_use_pct=80`
- `max_leverage=3`
- `max_daily_loss_pct=2`
- `max_consecutive_losses=3`
- `min_liquidation_distance_pct=10`

对应代码：`internal/risk/model.go:58` 到 `internal/risk/model.go:74`。

## 2. 关键风险缺口

### 2.1 日亏损和连续亏损停机未真正接入 paper 快照

`risk.AccountSnapshot` 支持 `DailyRealizedPnL` 和 `ConsecutiveLosses`，`risk.EvaluateOrder` 也会按这两个字段触发 `daily_loss_halt` 和 `consecutive_loss_halt`。

但当前 `paperRiskAccountSnapshot` 只传了权益、可用余额、敞口和保证金，没有传日内已实现盈亏和连续亏损次数。

对应代码：

- `internal/risk/model.go:77` 到 `internal/risk/model.go:88` 已定义账户快照字段。
- `internal/risk/model.go:150` 到 `internal/risk/model.go:158` 已有停机判断，并且不会阻止 reduce-only 平仓。
- `cmd/papertrade/positions.go` 会统计日内已实现 PnL 和连续亏损次数。
- `cmd/papertrade/positions.go` 会把这两个字段传入 `risk.AccountSnapshot`。

这意味着 paper 策略已经具备账户级熔断的第一版能力。真实交易前仍需要用交易所只读账户数据校准权益、可用余额和真实持仓风险。

### 2.2 单边行情仍缺完整趋势持仓模式

强单边行情里，策略现在可能出现两个问题：

- 过早止盈：ATR TP 到了就平，后续趋势主升段吃不到。
- 过早反转退出：短周期均线轻微反抽就触发 `trend_reversal`。

当前已经补了保本止损和 ATR trailing stop，但还没有：

- 分批止盈；
- 高周期趋势未破时继续持仓；
- 回踩后再入场；
- 趋势强度升高时自动放宽止盈上限。

这仍然会让策略偏短线 scalping；完整趋势跟随还需要分批止盈、剩余仓位持有规则和再入场规则。

### 2.3 防爆仓还缺真实交易所维度

当前强平价是本地估算，适合 paper 和预检查，但真实交易要以交易所返回为准：

- 真实可用余额；
- 真实持仓风险；
- 真实强平价；
- 真实维持保证金档位；
- 当前币对杠杆档位；
- 原生止盈止损保护单是否真的挂上。

本地估算不能替代交易所风控结果。以后接真实仓位时，必须以 OneBullEx 账户和持仓 API 为准。

### 2.4 回测还没有按行情状态拆分

当前回测会返回总收益、超额收益、最大回撤、交易次数和胜率，但没有拆成：

- 单边上涨行情；
- 单边下跌行情；
- 高波动震荡；
- 低波动震荡；
- 急涨急跌插针。

如果只看整体胜率，很容易被某段行情误导。跟单产品需要知道策略在哪类市场赚钱，在哪类市场必须停机。

## 3. 优化增强方案

### Phase 1：先补硬风控，禁止带缺口上真实交易

这是最高优先级。

1. 增加 paper trade log，记录每笔开仓、平仓、退出原因、持仓时长、净盈亏、手续费和滑点。
2. 用 trade log 计算 `DailyRealizedPnL` 和 `ConsecutiveLosses`，传入 `paperRiskAccountSnapshot`。
3. 增加账户级熔断：日亏损达到阈值后，当天只允许 reduce-only 平仓，不允许新开仓。
4. 增加连续亏损熔断：连续 3 笔亏损后暂停至少 N 根 K 线或当天暂停。
5. 原生保护单确认：真实下单后必须查询交易所，确认止盈止损保护已存在；没有保护单则立即撤单或 reduce-only 平仓。
6. 网络/API 异常时进入保护模式：行情过期、mark price 缺失、下单结果不确定时，不允许新开仓。

建议参数：

| 参数 | paper 小资金 | 真实灰度 | 说明 |
| --- | ---: | ---: | --- |
| `risk_pct` | `1% ~ 2%` | `0.3% ~ 0.8%` | 跟单资金必须按权益比例，不用固定数量 |
| `max_leverage` | `2x ~ 3x` | `1x ~ 2x` | 前期真实盘先保守 |
| `min_liquidation_distance_pct` | `15%` | `20% ~ 30%` | 距离强平越远，越不容易被插针打穿 |
| `max_margin_pct` | `25% ~ 35%` | `15% ~ 25%` | 限制总保证金占权益比例 |
| `max_daily_loss_pct` | `2%` | `1% ~ 2%` | 到线停机，不继续扛 |
| `max_consecutive_losses` | `3` | `2 ~ 3` | 连亏说明行情或信号失效 |

### Phase 2：增加高周期趋势分层

建议把策略拆成两个层级：

```text
高周期趋势层：1h / 15m，判断只做多、只做空、或者不交易
低周期执行层：5m / 3m，只负责找入场点和止损点
```

高周期趋势层建议输出 `trend_regime`：

- `strong_up`：只允许做多，禁止开空；
- `weak_up`：只允许轻仓做多；
- `range`：不交易或降低仓位；
- `weak_down`：只允许轻仓做空；
- `strong_down`：只允许做空，禁止开多。

判断指标建议先用规则，不急着上机器学习：

- `15m EMA20 > EMA60` 且 EMA20 斜率向上，偏多；
- `15m EMA20 < EMA60` 且 EMA20 斜率向下，偏空；
- `1h` 方向和 `15m` 方向一致时才提高仓位；
- ADX 或 ATR 百分位过低时视为震荡，不开仓；
- 价格距离高周期均线过远时不追单，等回踩。

落地后，当前 `fast/slow` 不再单独决定方向，只能在高周期允许的方向里触发入场。

### Phase 3：增加单边行情持仓机制

强单边行情应该从“固定 TP 到点就走”改成“先锁利润，再跟踪趋势”。

建议规则：

1. 到达 `1R` 后移动止损到开仓价附近，扣除手续费后至少不亏。
2. 到达 `1.5R` 或 `2R` 后平掉 `30% ~ 50%` 仓位。
3. 剩余仓位使用 ATR trailing stop，例如多单止损跟随 `最高价 - 1.2 * ATR`，空单止损跟随 `最低价 + 1.2 * ATR`。
4. 如果高周期趋势仍未反转，短周期快慢线轻微反向不立刻全平，只收紧 trailing stop。
5. 趋势强度很高时放宽 TP 上限，例如从当前 `1.40%` 提升到 `2.00% ~ 2.80%`，但必须经过回测验证。

这样可以兼顾短线交易频率和单边行情利润延展。

### Phase 4：跟单产品的资金保护

跟单产品不能只看主账户策略能不能赚钱，还要保护每个跟随账户。

必须增加这些约束：

- 每个 follower 单独计算可用权益、风险预算、最大保证金占用和最大日亏损。
- 主策略信号只提供方向和风险参数，不能直接复制固定数量。
- 跟随账户按自己的权益和止损距离重新计算下单数量。
- 每个 follower 都要有独立 kill switch。
- 主账户进入熔断时，所有 follower 禁止新开仓。
- 如果 follower 下单失败或保护单失败，不影响其他 follower，但必须记录失败原因。

跟单产品里最危险的是固定数量复制。正确方式是复制“信号 + 风险比例”，不是复制“下单数量”。

### Phase 5：回测和验证增强

必须补充以下回测能力：

1. 按市场状态拆分绩效：上涨趋势、下跌趋势、震荡、高波动、低波动。
2. 加入杠杆、保证金、强平价、资金费率和滑点模拟。
3. 做 walk-forward：用过去一段调参，用后一段验证，不允许只看同一段历史。
4. 每次参数版本入库，记录参数、回测区间、样本外区间、胜率、收益回撤比和最大连续亏损。
5. paper 连续运行至少 2 到 4 周，满足阈值后才能小资金真实灰度。

建议上线门槛：

- 样本外回测最大回撤低于 `8%`；
- paper 最大连续亏损不超过 `3`；
- paper 任意单日亏损不超过 `2%`；
- 交易所保护单确认成功率必须接近 `100%`；
- 网络异常和行情过期时没有新开仓记录。

## 4. 建议实施顺序

### 第一优先级：防爆仓和熔断

先做这些，因为它们不依赖策略是否优秀，但直接决定会不会出大事故：

1. 增加 trade log 和 `close_reason`。
2. 接入日内已实现盈亏和连续亏损统计。
3. 让 `paperRiskAccountSnapshot` 传入 `DailyRealizedPnL` 和 `ConsecutiveLosses`。
4. 加网络异常、行情过期和保护单缺失时的 hard halt。
5. 补测试：日亏损、连续亏损、强平距离不足、保护单缺失都必须拒绝新开仓。

### 第二优先级：趋势分层

然后做高周期趋势：

1. 支持读取 `15m` 和 `1h` K 线。
2. 增加 `trend_regime` 计算模块。
3. 信号入库时写入 `trend_regime`、高周期 EMA 斜率、ADX/ATR 状态。
4. 开仓前只允许顺着高周期方向交易。
5. 看板展示当前趋势状态和为什么不开仓。

### 第三优先级：单边行情利润延展

最后做收益增强：

1. 分批止盈。
2. 保本止损。
3. ATR trailing stop。
4. 强趋势模式放宽 TP。
5. 回踩再入场。

这部分会改变收益结构，必须回测和 paper 对比，不能一次性把所有逻辑打开。

## 5. 当前不建议做的事

- 不建议直接加高杠杆提高收益。高杠杆会放大插针、滑点和网络延迟风险。
- 不建议用机器学习直接决定买卖。现阶段数据量和标签质量不够，应该先用 ML 做信号过滤和参数选择。
- 不建议同时跑很多币对。先把 BTCUSDT 单策略跑稳定，再扩展 ETHUSDT 或其他高流动性币对。
- 不建议固定数量跟单。必须按每个账户权益和止损距离重新计算数量。

## 6. 最终目标

下一版策略应该从 `scalp-tpsl` 升级为：

```text
perp-trend-scalp-v3
```

核心变化：

- 高周期判断方向；
- 低周期找入场；
- ATR 动态止损；
- 分批止盈 + 保本止损 + trailing stop；
- 日亏损、连续亏损、强平距离和保护单确认硬熔断；
- 每个 follower 独立按权益和风险比例下单；
- 所有开仓、平仓、拒单、熔断原因可追踪、可回测、可复盘。

这套方向不能保证持续盈利，但能把策略从“会交易”提升到“可控风险地交易”。对跟单产品来说，先控制爆仓和大回撤，再追求更高胜率和更高频率。
